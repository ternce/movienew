# Исправление проблем с загрузкой видео - Итоговый отчёт

## 📋 Проблемы (исходные)

1. ❌ При создании шорта → загрузка видео → нажимаем "опубликовать" → ошибка "Новый контент не может быть опубликован без видео"
2. ❌ После нажимания "черновик" видео вроде сохраняется, но логика неверная
3. ❌ После публикации стоит бесконечная "Ожидание"
4. ❌ Захожу в сериалы → ошибка "Сериал не найден"
5. ❌ Захожу в клипы → бесконечная загрузка "Видео готовится к воспроизведению"

## ✅ Решение (реализовано)

### Корневая причина проблемы #1-3

**Неправильный flow:**
```
Загрузить видео в simple-mode → 
Создать контент с URL → 
Нажать "Опубликовать" → 
Backend проверяет VideoFile записи → 
НЕ находит → ошибка
```

**Правильный flow (реализовано):**
```
Создать контент как DRAFT → 
Получить contentId → 
Загрузить видео в content-mode (создается VideoFile) → 
Опубликовать контент
```

### Что изменилось

#### 1. Short Wizard (apps/web/components/studio/wizards/short-wizard.tsx)
- ✅ Добавлена двухшаговая логика:
  - Кнопка "Создать черновик" → создает контент как DRAFT
  - VideoUpload получает contentId и переходит в content-mode
  - Кнопка "Опубликовать" → обновляет контент на PUBLISHED
- ✅ Добавлены импорты: `useUpdateContent`, `toast`
- ✅ Видео теперь загружается через `/admin/content/:id/video/upload`

#### 2. Clip Wizard (apps/web/components/studio/wizards/clip-wizard.tsx)
- ✅ Трехшаговый процесс с автоматическим созданием черновика:
  - Шаг 1: Заполнить информацию
  - При переходе на Шаг 2: автоматически создается контент как DRAFT
  - Шаг 2: Загрузить видео (в content-mode)
  - Шаг 3: Выбрать возрастное ограничение
  - Финал: "Опубликовать" → обновить статус на PUBLISHED
- ✅ Улучшенная обработка ошибок

## 🔍 Если что-то не работает

1. **"Черновик не создается"**
   - Проверьте консоль браузера (F12)
   - Проверьте сетевые запросы (Network tab)
   - Убедитесь что все поля правильно заполнены

2. **"Видео не загружается"**
   - Проверьте размер файла (макс 200MB для shorts, 5GB для clips)
   - Проверьте формат видео (MP4, WebM)
   - Убедитесь что контент создан как DRAFT перед загрузкой

3. **"Контент не опубликовывается"**
   - Убедитесь что видео загрузилось и обработано (статус COMPLETED)
   - Проверьте логи backend (см. видение ниже)

## 📞 Техническая информация

### Видео Pipeline

```
Frontend: 
  1. создать содержание (DRAFT)
  2. загрузить видео

Backend:
  1. пол file на disk
  2. создать VideoFile запись (PENDING)
  3. очередь BullMQ job for transcoding
  4. transcoding запуск (PROCESSING)
  5. готовое видео (COMPLETED)
  6. обновить VideoFile статус
```

### API Endpoints

- `POST /api/v1/admin/content` - создание контента
- `PATCH /api/v1/admin/content/:id` - обновление контента
- `POST /api/v1/admin/content/:id/video/upload` - загрузка видео
- `GET /api/v1/admin/content/:id/video/status` - статус кодирования
